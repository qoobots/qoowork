# 开发指南

## 环境要求

| 依赖 | 版本要求 |
|------|----------|
| Node.js | `>= 24.15.0 < 25` |
| npm | 随 Node.js 提供 |
| Git | 用于 OpenClaw 子模块管理 |
| Python | Windows 打包时需要（`scripts/setup-python-runtime.cjs`） |

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/qoobots/qoowork.git
cd qoowork

# 2. 安装依赖
npm install

# 3. 首次运行：构建/同步 OpenClaw 运行时，然后启动应用
npm run electron:dev:openclaw

# 4. 日常开发：Vite (port 5175) + Electron 热加载
npm run electron:dev
```

## 常用命令

```bash
# ===== 开发 =====
npm run electron:dev           # 日常开发（需已完成 OpenClaw 运行时构建）
npm run electron:dev:openclaw  # 首次开发运行（含 OpenClaw 运行时构建同步）

# ===== 构建 =====
npm run build                  # 生产环境 Renderer 打包
npm run compile:electron       # 编译 Electron Main/Preload TypeScript

# ===== 测试 =====
npm test                       # 运行 Vitest 测试套件
npm test -- cowork             # 按名称过滤测试
npm test -- logger

# ===== 代码检查 =====
npm run lint                   # 全量 ESLint（可能因历史遗留问题失败）
npx eslint --ext ts,tsx --report-unused-disable-directives --max-warnings 0 <files>  # CI 等效检查

# ===== 打包 =====
npm run dist:mac               # macOS 打包
npm run dist:win               # Windows 打包
npm run dist:linux             # Linux 打包

# ===== OpenClaw 运行时管理 =====
npm run openclaw:runtime:host  # 构建当前平台 OpenClaw 运行时
npm run openclaw:patch         # 应用版本级 OpenClaw 补丁

# ===== Windows 环境配置 =====
npm run setup:mingit           # 设置 Windows 便携 Git
npm run setup:python-runtime   # 设置 Windows Python 运行时
```

## 开发工作流

### 1. 创建功能分支

基于 `master` 创建功能分支，命名使用 kebab-case：

```bash
git checkout -b feat/your-feature-name
# 或
git checkout -b fix/your-bug-name
```

### 2. 编写设计文档

在新功能或重要修复前，按 `specs/README.md` 规范创建设计文档：

```bash
# 创建 spec 目录
mkdir -p specs/features/your-feature-name

# 创建设计文档（YYYY-MM-DD-主题.md）
# 参考 specs/README.md 中的模板
```

### 3. 编写代码

遵循编码规范（详见下文），确保相关模块添加测试用例（`.test.ts` 文件）。

### 4. 本地验证

```bash
# 编译检查
npm run compile:electron

# 运行测试
npm test

# 代码检查（仅变更文件）
npx eslint --ext ts,tsx --report-unused-disable-directives --max-warnings 0 <your-changed-files>

# 启动应用手动验证
npm run electron:dev
```

### 5. 提交代码

使用 Conventional Commits 规范：

```
feat: 添加 XXX 功能
fix: 修复 YYY 问题
refactor: 重构 ZZZ 模块
docs: 更新文档
chore: 构建/工具链变更
```

## 编码规范

### 通用

- **语言**：TypeScript（严格模式）
- **缩进**：2 空格
- **引号**：单引号
- **分号**：必须
- **换行符**：LF
- **命名**：`PascalCase` 组件，`camelCase` 函数/变量，`kebab-case` 文件

### React 组件

- 使用函数式组件 + Hooks
- 使用 Tailwind CSS 进行样式处理
- Redux 状态切片命名：`*Slice.ts`
- 组件文件与同目录下的样式/测试文件就近放置

### 日志

通过 `src/main/logger.ts`（基于 `electron-log`）统一管理：

| 级别 | 使用场景 |
|------|----------|
| `console.error` | 需要排查的失败，附带 caught error |
| `console.warn` | 意外但可恢复的情况 |
| `console.log` | 有意义的生命周期事件 |
| `console.debug` | 高频或诊断细节 |

**禁止**：在轮询循环、每条消息的热路径、常规函数入口添加 info 级别日志。
日志消息使用英文，以模块标签开头，如 `[OpenClaw]`。

### IPC 通信

- 通道名称定义在 `src/shared/*/constants.ts`，避免硬编码
- Renderer → Main 必须通过 `contextBridge` 暴露的 API
- 禁止在渲染进程直接使用 Node.js API

## OpenClaw 集成要点

1. 当需要修改 OpenClaw 行为时，优先在 QooWork 侧寻找集成点（适配器、配置同步、插件配置、UI 处理）
2. 仅在无法通过 QooWork 侧实现时，才使用版本级 OpenClaw 补丁
3. 补丁存放于 `scripts/patches/<openclaw.version>/`
4. 不要将手动修改留在 OpenClaw 源码树中，应转换为补丁文件

## 环境变量

| 变量 | 说明 |
|------|------|
| `OPENCLAW_SRC` | 覆盖 OpenClaw 源码路径，默认 `../openclaw` |
| `OPENCLAW_SKIP_ENSURE=1` | 跳过自动 OpenClaw tag checkout |
| `OPENCLAW_FORCE_BUILD=1` | 强制重建 OpenClaw 运行时 |

## 常见问题

### Windows 下 `npm install` 失败

部分原生模块（如 `better-sqlite3`）需要 Python 和 C++ 编译工具链：

```bash
npm run setup:python-runtime
npm run setup:mingit
```

### OpenClaw 运行时未就绪

首次运行时需要构建 OpenClaw 运行时：

```bash
npm run electron:dev:openclaw
```

### 部分 ESLint 规则在全量检查中失败

项目存在历史遗留的 lint 债务。CI 仅检查变更文件。新增或修改的文件应通过：

```bash
npx eslint --ext ts,tsx --report-unused-disable-directives --max-warnings 0 <files>
```
