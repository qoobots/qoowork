# 测试指南

## 测试框架

QooWork 使用 [Vitest](https://vitest.dev/) 作为主测试框架。

```bash
npm test               # 运行全量测试
npm test -- cowork     # 按名称过滤
npm test -- logger     # 过滤特定模块
```

配置：`vitest.config.ts`，包含 `src/**/*.test.ts` 和 `tests/**/*.test.ts`。

## 测试策略

```
优先级：单元测试 > 集成测试 > E2E 测试
```

| 层级 | 覆盖范围 | 工具 | 频率 |
|------|----------|------|------|
| 单元测试 | 纯函数、工具类、共享逻辑 | Vitest | 每次提交 |
| 集成测试 | 模块交互、IPC 逻辑、数据层 | Vitest | 每次提交 |
| 手动验证 | UI 行为、Electron 交互、OpenClaw 集成 | `npm run electron:dev` | 发布前 |

## 编写测试

### 基本结构

```ts
import { describe, expect, test } from 'vitest';

describe('moduleName', () => {
  test('should do something', () => {
    const result = myFunction(input);
    expect(result).toBe(expectedValue);
  });
});
```

### 测试文件位置

- **推荐**：与源码同目录，命名为 `*.test.ts`
  ```
  src/renderer/services/cowork.test.ts
  src/main/libs/logger.test.ts
  ```
- **传统**：`tests/` 目录下的 `*.test.mjs` 文件使用 Node 内置 `node:test`，**不在默认 `npm test` 范围内**，仅需显式运行时使用

### 避免引入 Electron API

测试运行在 Node.js 环境，不加载 Electron。如需测试依赖 Electron API 的逻辑：

1. **内联提取**：将纯逻辑从 Electron 依赖中提取为独立函数
2. **依赖注入**：通过接口抽象 Electron API，测试时注入 mock
3. **模块隔离**：将可测试的部分放到不依赖 `electron` 的独立文件中

```ts
// ❌ 避免：测试中直接引用 electron
import { app } from 'electron';

// ✅ 推荐：提取纯逻辑
export function formatLogMessage(level: string, message: string): string {
  return `[${level.toUpperCase()}] ${message}`;
}
```

## 不同场景的验证要求

| 变更类型 | 验证步骤 |
|----------|----------|
| 纯文档变更 | 无需运行测试 |
| Renderer/UI 变更 | `npm test` + 手动验证 `npm run electron:dev` |
| Main 进程/IPC/运行时变更 | `npm test -- <模块>` + `npm run compile:electron` 或 `npm run build` |
| OpenClaw 集成变更 | 验证运行时启动、配置同步、网关行为、相关日志 |
| Preload 变更 | `npm run compile:electron` + 手动验证 |

## GitHub CI

CI 流程（`.github/workflows/`）：
1. 运行 `npm test`（Vitest 全量）
2. 对变更的 TypeScript 文件运行 ESLint（非全量）

## 遗留测试

```
tests/*.test.mjs
```

这些是使用 Node 内置 `node:test` 编写的旧版测试，不在默认 `npm test` 运行范围内。仅按需显式执行：

```bash
node --test tests/<file>.test.mjs
```

如需覆盖相关逻辑，建议迁移为 Vitest 格式（`.test.ts`）并与源码同目录放置。

## 测试数据

- 测试数据尽量使用内存 Mock，不依赖外部服务
- SQLite 测试可使用 `:memory:` 模式或临时文件
- 避免在测试中硬编码绝对路径，使用 `os.tmpdir()` 或 Vitest `tmpdir` fixture

## 调试测试

```bash
# 监听模式
npm test -- --watch

# 单文件运行
npx vitest run src/path/to/file.test.ts

# 带 UI 运行
npx vitest --ui
```
