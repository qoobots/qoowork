# IM 渠道集成

## 概述

QooWork 通过 OpenClaw 插件体系支持多渠道 IM 接入，允许 Agent 通过钉钉、飞书、企业微信、微信、QQ、Discord 等即时通讯平台与用户交互。

## 支持的渠道

| 渠道 | OpenClaw 插件包 | 插件 ID | 状态 |
|------|----------------|---------|------|
| 钉钉 | `@dingtalk-real-ai/dingtalk-connector` | `dingtalk-connector` | 正式 |
| 飞书/Lark | `@larksuite/openclaw-lark` | `openclaw-lark` | 正式 |
| QQ Bot | `@openclaw/qqbot` | `qqbot` | 正式 |
| Discord | `@openclaw/discord` | `discord` | 正式 |
| 企业微信 | `@wecom/wecom-openclaw-plugin` | `wecom-openclaw-plugin` | 正式 |
| 微信 | `@tencent-weixin/openclaw-weixin` | `openclaw-weixin` | 正式 |
| POPO | `moltbot-popo` | `moltbot-popo` | 可选 |
| 网易云信 (NIM) | `openclaw-nim-channel` | `openclaw-nim-channel` | 可选 |
| 网易蜂巢 (Bee) | `openclaw-netease-bee` | `openclaw-qoobot-bee` | 可选 |

## 架构

```
QooWork Main Process
├── src/main/im/
│   ├── IM 配置 CRUD (im_config 表)
│   ├── IM 状态管理 (启停/连接状态)
│   ├── IM 消息投递 (发送/接收路由)
│   ├── IM 会话映射 (IM 对话 → Cowork 会话)
│   └── IM 媒体处理 (图片/文件)
│
└── OpenClaw 运行时
    └── 各渠道插件 (作为 OpenClaw 网关插件运行)
        ├── 消息接收与解析
        ├── Agent 响应格式化
        └── 渠道认证与连接维护
```

## 数据模型

### im_config 表

存储各 IM 渠道的配置信息，包括：
- 渠道类型（dingtalk / lark / wecom / weixin / qqbot / discord）
- Bot Token / App ID / Secret 等认证信息
- Webhook URL / 回调地址
- 启用状态
- 多实例标识

### im_session_mappings 表

IM 对话到 Cowork 会话的映射：
- IM 平台 + 频道/群组 ID
- 对应的 Cowork Session ID
- 关联的 Agent ID
- OpenClaw Session Key

## 核心流程

### 消息接收

```
IM 用户发消息
  → 渠道插件接收
    → OpenClaw 路由到指定 Agent
      → Agent 生成响应
        → 渠道插件格式化并回复
```

### 会话映射

每次 IM 对话与 Cowork 会话通过 `im_session_mappings` 建立关联，确保：
- 同一 IM 对话的上下文连续
- 支持多 Agent 绑定（不同群/频道可绑定不同 Agent）
- 支持会话切换与重置

### 多实例支持

部分渠道（如 Telegram、企业微信）支持在同一平台创建多个实例，通过独立的 Bot Token 或 App 配置区分。

## 新增渠道指南

### 1. 引入 OpenClaw 插件

在 `package.json` → `openclaw.plugins` 添加插件声明：

```json
{
  "id": "new-channel",
  "npm": "@vendor/openclaw-new-channel",
  "version": "x.y.z"
}
```

### 2. 添加配置管理

在 `src/main/im/` 添加渠道配置处理逻辑，包括：
- 配置表单数据结构
- SQLite CRUD 操作
- 状态管理

### 3. 添加 UI 界面

在 `src/renderer/components/im/` 添加渠道配置 UI 组件。

### 4. 同步到 OpenClaw

在 `src/main/libs/openclawConfigSync.ts` 中扩展配置同步逻辑。

### 5. IPC 通道

在 `src/shared/im/` 定义相关 IPC 通道常量和类型。

## 调试
