# 国际化

## 概述

QooWork 支持中英文双语界面。国际化方案基于自定义 i18n 模块，非第三方 i18n 库。

## 核心模块

位置：`src/renderer/services/i18n.ts`

提供：
- i18n 字典（中英文键值对）
- `t()` 翻译函数
- 当前语言状态管理

## 翻译函数

```ts
import { t } from '@/renderer/services/i18n';

// 使用翻译键
const label = t('settings.general.theme');

// 支持参数替换
const message = t('welcome.message', { name: userName });
```

## 语言切换

用户在设置中切换语言后：
1. 更新 Redux 中语言状态
2. 所有通过 `t()` 渲染的文本自动更新
3. 语言偏好持久化到 SQLite

## 翻译词典维护

### 结构

```ts
const zhDict = {
  settings: {
    general: {
      theme: '主题',
      language: '语言',
    },
  },
};

const enDict = {
  settings: {
    general: {
      theme: 'Theme',
      language: 'Language',
    },
  },
};
```

### 新增翻译

1. 在 `zhDict` 中添加中文条目
2. 在 `enDict` 中同步添加英文条目
3. 使用相同键路径

## 规范

- **翻译键**：使用点号分隔的小驼峰路径，如 `settings.general.theme`
- **文案**：中文使用全角标点，英文使用半角标点
- **命名**：键名使用英文描述，方便开发者理解
- **技术术语**：保留英文，不翻译
- **换行**：长文本使用模板字符串

## 注意事项

- 不要在代码中硬编码 UI 文本，始终使用 `t()` 函数
- 新增功能时同步添加中英文翻译
- README 和文档类内容使用独立的中英文版本（`README.md` / `README_zh.md`）
- 开发者日志和注释使用英文
