# Skills 技能系统

## 概述

Skills 是 QooWork 的内置技能扩展系统，为 Agent 提供专业化的领域能力。每个 Skill 包含 `SKILL.md` 入口文件，定义其使用场景、触发条件和执行指令。

## Skill 结构

每个 Skill 目录包含：

```
skill-name/
├── SKILL.md               # 入口文件（必需）：描述、触发条件、指令
├── scripts/               # 辅助脚本（可选）
├── resources/             # 资源文件（可选）
└── package.json           # 依赖声明（可选）
```

### SKILL.md 格式

```markdown
# Skill Name

简要描述技能的功能和应用场景。

## 触发条件

- 用户明确提到相关场景
- 特定关键词触发

## 指令

详细的执行指令和工作流程。
```

## 生命周期管理

### 配置 (skills.config.json)

```json
{
  "skills": [
    {
      "id": "pdf",
      "enabled": true,
      "order": 1
    }
  ]
}
```

### 安装与升级

由 `src/main/skillManager.ts` 管理：
- **同步**：启动时将内置 Skills 同步到用户目录
- **安装**：用户通过 UI 安装新 Skill
- **升级**：检测版本更新并提示升级
- **安全扫描**：安装前对 Skill 进行安全检查
- **启用/禁用**：用户可自由开关各 Skill

### Skills 运行时

- 安装后，Agent 的 System Prompt 会注入可用 Skills 的描述
- 触发时，Skill 的 `SKILL.md` 内容会作为上下文注入当前会话
- 支持 Skills 间组合调用（如 `pdf` + `xlsx` 处理文档中的表格数据）

## 开发新 Skill

### 1. 创建目录

```bash
mkdir -p SKILLs/your-skill-name
```

### 2. 编写 SKILL.md

按照上文格式编写入口文件，确保：
- 清晰描述技能功能
- 明确触发条件
- 详细执行指令

### 3. 添加辅助资源（可选）

- `scripts/` 下的可执行脚本
- `resources/` 下的模板、配置等
- `package.json` 声明 Node.js 依赖

### 4. 注册到配置

在 `skills.config.json` 添加 Skill 声明。

### 5. 测试

1. 将 Skill 放入 `SKILLs/`
2. 重启应用
3. 在 Skill 管理中确认可见并启用
4. 通过对话触发测试

## 安全原则

- Skill 安装前进行安全扫描
- 用户手动启用后方可被 Agent 调用
- Skill 脚本运行在受限环境中
- 敏感操作需用户确认
