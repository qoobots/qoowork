# 构建与发布

## 构建流程

### 本地构建

```bash
# 1. 编译 Electron Main/Preload TypeScript
npm run compile:electron
# 产物 → dist-electron/

# 2. 构建 Renderer (Vite 生产打包)
npm run build
# 产物 → dist/

# 3. 完整构建（compile + build）
npm run compile:electron && npm run build
```

### OpenClaw 运行时构建

首次开发或运行时变更时：

```bash
# 构建当前平台 OpenClaw 运行时
npm run openclaw:runtime:host

# 应用版本级补丁
npm run openclaw:patch
```

构建脚本链：
1. `scripts/ensure-openclaw-version.cjs` — 克隆/拉取/检出指定版本
2. `scripts/apply-openclaw-patches.cjs` — 应用版本级补丁
3. `scripts/run-build-openclaw-runtime.cjs` — 构建平台运行时
4. `scripts/sync-openclaw-runtime-current.cjs` — 链接 `vendor/openclaw-runtime/current`
5. `scripts/bundle-openclaw-gateway.cjs` — 创建网关 bundle
6. `scripts/ensure-openclaw-plugins.cjs` — 安装第三方插件
7. `scripts/sync-local-openclaw-extensions.cjs` — 同步本地扩展
8. `scripts/precompile-openclaw-extensions.cjs` — 预编译扩展
9. `scripts/install-openclaw-channel-deps.cjs` — 安装渠道依赖
10. `scripts/prune-openclaw-runtime.cjs` — 清理无用内容

运行时产物目录：`vendor/openclaw-runtime/`（gitignore）

### 渠道归属信息

构建时自动生成 `.keyfrom-build/keyfrom.json`（gitignore）：

```bash
scripts/generate-keyfrom-build-info.cjs
```

## 平台打包

使用 [electron-builder](https://www.electron.build/)，配置位于 `electron-builder.json`。

```bash
npm run dist:mac      # macOS (.dmg)
npm run dist:win      # Windows (NSIS 安装包)
npm run dist:linux    # Linux (.AppImage / .deb)
```

### Windows 打包注意事项

```bash
# 设置便携 Git（如需）
npm run setup:mingit

# 设置 Python 运行时
npm run setup:python-runtime

# 打包
npm run dist:win
```

打包产物会自动包含：
- OpenClaw 运行时（`Resources/cfmind/`）
- Node.js 运行时（`resources/node-runtime/`）
- 系统托盘图标（`resources/tray/`）

### 签名

应用签名配置在 `electron-builder.json` 中。构建资源（图标、entitlements）位于 `build/`：

```
build/
├── icons/
│   ├── mac/icon.icns
│   └── win/icon.ico
├── entitlements.mac.plist
└── ...
```

## 版本管理

版本号定义在 `package.json` → `version` 字段，使用格式 `YYYY.M.D`（如 `2026.7.3`）。

OpenClaw 版本在 `package.json` → `openclaw.version`，格式 `vYYYY.M.D`。

### 升级 OpenClaw

1. 更新 `package.json` 中 `openclaw.version`
2. 审核并更新 `scripts/patches/` 下对应版本的补丁
3. 运行 `npm run electron:dev:openclaw` 验证
4. 本地测试通过后提交

## 发布检查清单

发布前确认以下项目：

- [ ] 所有 vitest 测试通过：`npm test`
- [ ] ESLint 检查变更文件通过
- [ ] `npm run compile:electron` 编译无误
- [ ] `npm run build` 构建成功
- [ ] 目标平台打包成功（`dist:mac` / `dist:win` / `dist:linux`）
- [ ] 手动验证核心功能：会话、IM、Skills、Artifacts
- [ ] OpenClaw 网关正常启动、配置同步无误
- [ ] `docs/server-integration/` 中相关 API 变更已记录
- [ ] `specs/` 设计文档已更新
- [ ] CHANGELOG（如有）已更新
- [ ] release notes 已准备

## 回滚方案

当发布版本出现严重问题时：

1. **代码回滚**：`git revert` 回退有问题的提交，重新打包
2. **应用降级**：引导用户安装上一个稳定版本的安装包
3. **服务端降级**：如需配合服务端变更，同步回滚服务端 API

## 产物清理

```bash
# 清理构建产物
rm -rf dist/ dist-electron/

# 清理 OpenClaw 运行时
rm -rf vendor/openclaw-runtime/
```
